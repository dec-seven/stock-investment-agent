# 方法论更新器 Skill

## 概述

本 SKILL 的核心职责是 **学习方法论并更新知识库**。

**核心定位：**
> 从样例中提取分析方法论 → 择优录取 → 更新知识库

---

## 知识库架构

```
knowledge-bases/
├── stock-methodology/                   # 方法论知识库
│   ├── stock_morning_brief_guide.md     # 早报分析方法论主文件
│   ├── stock_morning_brief_templates.md # 早报模板参考
│   ├── stock_selection_guide.md         # 选股方法论
│   └── manifest.json                    # 元数据与更新日志
│
└── stock-samples/                   # 样例存储（不对外）
    ├── morning_brief/               # 早评/盘前分析样例
    │   ├── 2026-06-09.txt
    │   ├── 2026-06-14_weekend_message_method_sample.md
    │   └── ...
    └── stock_selection/             # 选股样例
        ├── 2026-06-14_weekend_message_selection_method_sample.md
        └── ...
```

## 工作流程

### Phase 1: 样例摄入

```
1. 识别样例类型
   ├─ 早评/盘前分析样例 → 保存到 stock-samples/morning_brief/YYYY-MM-DD[_主题].md
   └─ 选股样例 → 保存到 stock-samples/stock_selection/YYYY-MM-DD[_主题].md

   重要：同一份学习素材若同时包含盘前分析方法和选股方法，应拆分或复制保存到两个目录，分别记录提炼结果。

2. 提取样例元数据
   ├─ 样例日期
   ├─ 市场状态（上涨/下跌/震荡）
   ├─ 核心观点
   └─ 关键判断
```

### Phase 2: 方法论提取

**分析方法论提取：**

```
解剖维度：
1. 市场定位层 → 这份样例如何判断当前市场阶段？
2. 信号判断层 → 使用了哪些见底/见顶信号？
3. 逻辑挖掘层 → 如何分析板块涨跌的根本原因？
4. 外部关联层 → 如何关联海外/政策/事件？
5. 操作策略层 → 仓位建议的依据是什么？
```

**选股方法论提取：**

```
解剖维度：
1. 题材识别 → 如何识别热点题材？
2. 催化逻辑 → 催化优先级如何排序？
3. 个股筛选 → 如何从题材中筛选个股？
4. 时机判断 → 买入时机的判断标准？
```

### Phase 3: 择优录取

```
现有方法论 vs 样例方法论
     │
     ├── 样例更优 → 更新方法论
     │
     └── 现有更优 → 保持不变
```

### Phase 4: 更新知识库

```
1. 读取现有方法论文件
   ├─ stock_morning_brief_guide.md
   ├─ stock_morning_brief_templates.md
   └─ stock_selection_guide.md

2. 合并新方法论（择优录取）

3. 更新文件并记录更新日志
   ├─ 更新方法论文件
   └─ 在 manifest.json 中添加更新记录
```

## 更新策略

| 场景 | 更新方式 | 说明 |
|------|---------|------|
| 新增分析维度 | 直接添加 | 在现有框架中补充新维度 |
| 优化现有方法 | 替换更新 | 样例方法更优时替换 |
| 发现反例 | 标注限制 | 记录方法的适用边界 |
| 新增策略类型 | 新增章节 | 扩充方法论内容 |

---

## 注意事项

1. **此 SKILL 不对外分发**：包含方法论学习逻辑，仅自己使用
2. **持续更新**：每次有新案例时直接更新方法论，保持知识的时效性
3. **保留样例**：样例是学习方法论的来源，持续积累；更新方法论前必须先保存原始学习素材到 `knowledge-bases/stock-samples/` 对应目录
4. **记录更新**：每次更新在 manifest.json 中记录变更内容
